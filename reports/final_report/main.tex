\section*{1. Overview}
Aussie EcoLens is a multi-cloud online media system for wildlife observation data. Authenticated users can upload camera-trap images and short videos, receive automatic species tags from the supplied course machine-learning model, search media by tags or species, retrieve a full image from a thumbnail URL, manually add or remove tags, delete media, and subscribe to watched-tag notifications. The final deployment uses AWS for authentication, API, storage, database, video frame extraction, and notifications, while Google Cloud hosts the ML inference service and mirrored metadata. This split keeps the application serverless while allowing the large PyTorch, MegaDetector, and classifier runtime to run in a suitable container service.

\section*{3. Design and Implementation Choices}
The system is serverless because the workload is event-driven: uploads, queries, tag edits, deletes, and notification watches happen intermittently and should scale without managing virtual machines. Deduplication uses a SHA-256 checksum, so repeated uploads return existing metadata instead of wasting storage. Images receive compressed thumbnails with preserved aspect ratio. Videos are sampled at one frame per second with ffmpeg; each extracted frame is submitted to the same model service and the predictions are aggregated as tag counts. Model handling is flexible: the GCP service reads classifier and detector blob paths from environment variables, so a future model version can be swapped by changing Cloud Storage object names rather than changing Lambda or Cloud Run source code.

\section*{4. Core Requirement Coverage}
\begin{summarybox}{Implemented and Verified}
\begin{itemize}
  \item Authentication and authorisation: AWS Cognito Hosted UI plus API Gateway JWT authoriser; unauthenticated API calls return 401.
  \item File handling: S3 originals, S3 thumbnails and video frames, checksum deduplication, DynamoDB metadata.
  \item ML tagging: GCP Cloud Run runs MegaDetector \texttt{mdv5a.pt} and the supplied classifier \texttt{model.pt}.
  \item Queries: tag-count AND query, species query, thumbnail-to-full-image lookup, query-by-file without permanent storage.
  \item Mutations: bulk manual tag add/remove and deletion of originals, thumbnails, frames, and metadata.
  \item Notifications: watched-tag events create in-app records and publish through SNS and verified SMTP email.
\end{itemize}
\end{summarybox}

\section*{5. User Guide for Testing}
\begin{enumerate}[leftmargin=*]
  \item Start the UI with \texttt{python3 -m src.aussie\_ecolens.server} and open \texttt{127.0.0.1:8080}.
  \item Register or sign in through Cognito. After login, the app shows the user's name and protected tools.
  \item In Image mode, upload one or more test images. Expected output: thumbnails, model-derived tag chips, duplicate feedback, and copyable Thumbnail URLs.
  \item Query by tag count, for example species \texttt{casuarius\_casuarius} with minimum count 1, then repeat as a species-only query.
  \item Paste a Thumbnail URL into the thumbnail lookup tool. Expected output: an openable full original image URL.
  \item Use Query by file to submit a file for temporary tag detection. The file is not stored as new media.
  \item Add or remove manual tags from one or more media URLs, then query the tag to verify the update.
  \item Switch to Video mode and upload a short video. Expected output: original video URL, extracted frame previews, and aggregated model tags.
  \item Register a watch for a species tag, upload matching media, refresh Notifications, and confirm the SMTP email.
  \item Delete a media URL and verify it disappears from query results and storage-derived outputs.
\end{enumerate}
